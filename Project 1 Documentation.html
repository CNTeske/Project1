Project 1: Corey Schotte & Christian Teske

Waterfall Development Process:

  1. Requirements

    • Ordinary users can: log in, see their requests' status, make new requests, log out.
    • Finance Managers can: log in, see all requests, filter requests, approve/disaprove requests, log out.
    • AWS Database: holds user/manager account info, adds requests from users, updates requests via managers
    • Manager/User Account info: User ID, username, password, first name, last name, email. Accounts and roles are final in DB.
    • Requests info: Request ID, Submitting User ID, Amount, Description, Recipt picture, Status, Time submitted, Time resolved, Resolver name.

  2. Analysis

    • SQL database in Postgres to store data, with a JDBC connection driver to access the database.
    • Angular produced single-page application using AJAX to call server components.
    • Password encription via Java, to be stored in the database.
    • Picture uploading and storage on the database.
